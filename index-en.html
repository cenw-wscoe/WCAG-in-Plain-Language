Does every page need an <h1>? Not exactly. A heading cannot be skipped, meaning that if your page contains an <h2> and it's immidiate subheading is an <h4>, that's a failure.
However, a page doesn't absolutely need an <h1>, nor does it need to start with an <h1>. Although it is not advised, pages can start with an <h2>, for example.		 1.3.1 - Info and relationship (Level A) False No

Is the use of autocomplete="" obligatory on input fields? For input fields that are repetitive across the web (email, password, birthday, etc), yes. However, it can be set to "off" on those elements if desired.
If the input field is app or site specific, such as "Enter your PUI code here", the field doesn't require the autocomplete attribute. However, it still is recommended.		 1.3.5 - Identify Input Purpose (Level AA) Maybe No
